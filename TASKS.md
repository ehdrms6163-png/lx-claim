
## 작업 5: 주소록 xlsx 연동

### 파일 구조 (주소록전화번호_삭제.xlsx)
- Sheet1, 60행 x 11열
- 4행: 헤더 (팀, 물류, 업무구분, 협력업체, 대표자, 대표번호, 팩스번호, 주소, 라인팀LM, LM메일, 라인팀팀장)
- 5행: 서브헤더 (9열이 "라인팀 LM"으로 분리)
- 6행~: 데이터 (팀명은 병합셀로 첫 행에만 있음)

### 파싱 규칙
- A열(팀): 값이 있으면 현재 팀, 없으면 이전 팀 유지 (병합셀)
- B열(물류): 값이 있으면 현재 물류센터
- C열(업무구분): "물류", "에어컨", "B2B" 등 업무구분
- D열(협력업체): BP사명
- I열(라인팀LM): LM 이름 + 연락처 (줄바꿈으로 구분)
- J열(LM메일): LM 이메일
- K열(라인팀팀장): 팀장 이름 + 연락처

### 저장 구조
```js
// Firestore: cfg_addressbook 컬렉션
{
  lineTeam: "수도권서부팀",
  logistics: "부천물류",        // B열 물류명
  bizType: "부천물류",          // C열 업무구분
  bpName: "위더스로지스",        // D열 협력업체
  lmName: "기영연 책임",         // I열에서 이름 추출
  lmEmail: "youngyoun.ki@lxpantos.com",  // J열
  teamLeader: "이정렬 팀장"      // K열에서 이름 추출
}
```

### 구현 내용
1. 설정관리에 "주소록 관리" 탭 추가
2. xlsx 파일 업로드 버튼 → 파싱 후 Firestore `cfg_addressbook`에 저장
3. 저장된 주소록을 표로 표시 (팀/물류/BP사/LM/팀장)
4. 신규접수 폼에서 "물류" 입력 시 해당 물류의 LM 정보 자동 표시 (참고용)
5. 전역변수 `addressbook = []` 로드 추가

### 주의사항
- SheetJS로 파싱 (이미 로드돼 있음)
- I열 LM 정보는 줄바꿈(\n)으로 "이름\n연락처" 형태 → 이름만 추출
- K열 팀장 정보도 동일하게 이름만 추출
